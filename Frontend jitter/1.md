## VSCode LaTeX 配置指南

### 第一步：安装 LaTeX 发行版

根据你的操作系统选择：

| 系统 | 推荐发行版 | 安装方式 |
|------|-----------|----------|
| **Windows** | MiKTeX | 下载安装包：https://miktex.org/download |
| **macOS** | MacTeX | `brew install --cask mactex` 或下载：https://tug.org/mactex/ |
| **Linux** | TeX Live | `sudo apt install texlive-full` (Ubuntu/Debian) |

> ⚠️ **注意**：完整安装约 4-7GB，建议选择完整版避免缺包问题

---

### 第二步：安装 VSCode 扩展

打开 VSCode，按 `Ctrl+Shift+X`（Mac: `Cmd+Shift+X`），搜索并安装：

**必装：**
```
LaTeX Workshop
```
作者：James Yu，下载量最高的那个

**可选（推荐）：**
```
LTeX – LanguageTool grammar/spell checking
```

提供语法检查

### 第三步：基本使用

#### 1. 创建文件

新建文件，保存为 `.tex` 后缀，比如 `paper.tex`

#### 2. 编译方法

有三种方式：

**方法 A：快捷键**
```
Ctrl + Alt + B  (Windows/Linux)
Cmd + Option + B  (Mac)
```

**方法 B：保存时自动编译**
默认开启，保存文件就会自动编译

**方法 C：点击按钮**
点击右上角的绿色播放按钮 ▶️

#### 3. 预览 PDF

```
Ctrl + Alt + V  (Windows/Linux)
Cmd + Option + V  (Mac)
```

或者点击左侧边栏的 TEX 图标 → "View LaTeX PDF"

### 常见问题

#### ❌ 编译失败：缺少包

**MiKTeX**：会弹窗提示安装，点"是"即可

**TeX Live**：手动安装
```bash
sudo tlmgr install <package-name>
```

#### ❌ 中文显示问题

如果你要写中文，需要：

1. 文件开头改成：
```latex
\documentclass[11pt,a4paper]{article}
\usepackage{ctex}  % 添加这行
```

2. 使用 XeLaTeX 编译：
   - 按 `Ctrl + Shift + P`
   - 输入 `LaTeX Workshop: Build with recipe`
   - 选择 `xelatex × 2`

#### ❌ PDF 预览空白

点击 VSCode 左下角的 "OUTPUT"，选择 "LaTeX Workshop"，查看错误信息

作为学生个人发表，建议遵循以下路径：**实验 -> 完善论文 -> ArXiv (占坑/公开) -> 会议/期刊 (正式发表)**。

#### 2. 发布到 ArXiv (预印本)
ArXiv 是物理、计算机领域的预印本服务器，发在这里代表“我先做出来了”。

*   **注册**：去 [arxiv.org](https://arxiv.org/) 注册账号（最好用学校 `.edu` 邮箱，审核通过率高）。
*   **Endorsement (背书)**：
    *   这是学生最难的一步。因为你是新用户，ArXiv 需要有人“担保”你的论文不是垃圾。
    *   **解决方法**：系统会提示你需要 `cs.HC` (Human-Computer Interaction) 或 `cs.PF` (Performance) 领域的背书。你需要找到在这个领域发过 ArXiv 的人（可以是你的导师、认识的教授，或者礼貌地发邮件给在该领域活跃的学者），请求他们点击背书链接。
*   **提交**：
    *   ArXiv 不接受 PDF，只接受 **LaTeX 源码**。
    *   你需要把 paper.tex, sections, `figures/` 打包成 `.tar.gz` 或 `.zip` 上传。
    *   确保你的 `.tex` 文件里没有绝对路径（如 `D:\resource\...`），图片引用要用相对路径。

#### 3. 投稿到期刊/会议 (正式发表)
ArXiv 只是存档，不算正式出版物。你需要投给同行评审的会议或期刊。

*   **推荐去向 (前端/Web方向)**：
    *   **顶级会议**：The Web Conference (WWW) - 难度极高。
    *   **专业会议**：ICWE (Intl. Conf. on Web Engineering) - 难度适中，适合技术类论文。
    *   **期刊**：
        *   *IEEE Software* (如果侧重工程实践)
        *   *ACM Transactions on the Web (TWEB)*
    *   **学生友好**：许多会议有 "Student Track" 或 "Poster/Demo Track"，只需要写 2-4 页，录用率较高，非常适合展示你的 Demo。

*   **投稿流程**：
    1.  去会议官网看 "Call for Papers"。
    2.  下载官方 LaTeX 模板（通常是 `acmart` 或 `IEEEtran`），把你的内容复制进去重新排版（这一步很繁琐但必须做）。
    3.  在截稿日期前提交 PDF。
    4.  等待 2-3 个月的 Peer Review。
